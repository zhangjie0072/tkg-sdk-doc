In App Purchase

danger
  • Add USE_IAP macro definition
  • Unity In App Purchasing plug-in version 4.9.3 or later

In App Purchase access method​

1、Import IAP Plugin​

Import Unity In App Purchasing Plugin.
Unity menu bar -> Window -> Package Manager -> In App Purchasing -> Install.

2 Add macro definition​

USE_IAP

3、Add Product IDs​

tip

Support two ways:
a. In IAPProducts.cs, pre-set the product IDs.
b. Adding Product IDs through the AddProducts Interface.

Note: The product type you add must match the type requested in the backend application. Otherwise, it may not be able to retrieve the corresponding product information. 
**ProductType.NonConsumable**  only buy once. Good for one-time purchases, such as extra levels.        
**ProductType.Consumable**     can be purchased repeatedly. Suitable for consumable goods such as virtual currency.        
**ProductType.Subscription**   can be purchased and restored repeatedly. Durable goods, but for a limited period (subscription).

3.1 In IAPProducts.cs, pre-set the product IDs.​

Add item ID in Assets/TKGSDK/Config/Scripts/IAPProducts.cs file.

public static class IAPProducts
{
    /// <summary>
    /// Item ID
    /// </summary>
    public const string NoAds = "test.tremoveads";

    public readonly static Dictionary<string, ProductType> ProductDic = new Dictionary<string, ProductType>()
    {    
        //ProductType.NonConsumable only buy once. Good for one-time purchases, such as extra levels
        //ProductType.Consumable can be purchased repeatedly. Suitable for consumable goods such as virtual currency
        //ProductType.Subscription can be purchased and restored repeatedly. Durable goods, but for a limited period (subscription)
        { NoAds, ProductType.NonConsumable},
        //Add more product
        ...
        ...
    };
}

3.2 Adding Product IDs through the AddProducts Interface. (v3.1.2new)​

TKGSDKManager.Instance.AddProducts(addProDic);

API:

void AddProducts(Dictionary<string, ProductType> products);
NameisRequiredTypeDescription
productsYDictionary<string, ProductType>Products IDs

ex:

Dictionary<string, ProductType> ProductDic = new Dictionary<string, ProductType>()
{
    { "com.tkkk.unitysdk.demo.1",ProductType.Consumable},
    { "com.tkkk.unitysdk.demo.a1",ProductType.Consumable},
    { "com.tkkk.unitysdk.demo.a12",ProductType.Consumable}
};

TKGSDKManager.Instance.AddProducts(ProductDic);

5、In App Purchase (v3.1.2new)​

TKGSDKManager.Instance.BuyProductByID(productID, productName, gameExtra);

API:

void BuyProductByID(string productId, string enProductName, string gameExtraParam = "")
NameisRequiredTypeDescription
productIdYstringProduct ID
GooglePlay：ProductID iOS：SKU
enProductNameYstringProduct Name
Custom product name on the game side
gameExtraParamNstringGame Extension Field
Passed from the game to the SDK in the payment interface and returned unchanged in the payment callback.

ex:

public void BuyProduct()
{
    string productID = "com.tkkk.unitysdk.demo.a1";
    string productName = "a1";
    string gameExtra = "a1GameExtraParam";
#if USE_IAP
    TKGSDKManager.Instance.BuyProductByID(productID, productName, gameExtra);
#endif
}

In-purchase exception​

In case of payment failure, please confirm the following issues:

  • All product categories need to be consistent with the store configuration, consumable, non-consumable or subscription products
  • If the account in China cannot be used for payment, you need to switch regions or use another region account
  • Network reasons, trying to switch between different vpn nodes

Server Integration (Optional)(v3.1.2 new)​

After a successful payment through the SDK, the game retrieves the payment result via a callback. For games with a server, it is recommended to prioritize integration with the SDK server's payment callback interface. After a successful payment, the SDK server will notify the game server to deliver the purchased goods. In the case of standalone games, ensure that the goods are delivered to the player after receiving a successful payment callback from the client.

Preparation for third-party game callback​

1、Obtain game-side authorization encryption secret from the integration partner.
2、Provide the game-side callback URL to the integration partner for configuration.

Game-side Callback Address Interface Description​

1、Request Method: Post
2、Header Authorization: Authorization: secretkey
3、Body Json

4、Encryption Method
Calculate the value of the "Sign" field by concatenating the parameter values (parameter value 1 + parameter value 2 + parameter value n + secret key) in the order of the returned parameters, and then applying MD5 encryption. The "+" sign is used as a connector and is not included in the calculation.

$sign = md5("qa"+"android"+"com.test.demo"+"0cc852594cb2dfd0381af38f687b44906"+"com.tkkk.unitysdk.demo.a1"+"GPA.3340-7674-3284-90321"+"1.99"+"USD"+"game_extra_param_value"+"1692346624"+$secretKey)

5、Retry Mechanism
Retry up to 10 times within a 5-hour timeframe with the following intervals:

Retry 1: After 35 seconds
Retry 2: After 40 seconds
Retry 3: After 115 seconds
Retry 4: After 150 seconds
Retry 5: After 385 seconds
Retry 6: After 755 seconds
Retry 7: After 1500 seconds
Retry 8: After 2610 seconds
Retry 9: After 4230 seconds
Retry 10: After 6850 seconds
Continue retrying until the returned HTTP status is 200 and the output is 'success'.
