In App Purchase

danger
  • Add USE_IAP macro definition
  • Unity In App Purchasing plug-in version 4.4.0 or later
  • If a Unity Gaming Services component is used in your project, ensure that it is manually initialized and that the SDK initialization method is executed in the Unity Gaming Services component initialization success callback

In App Purchase access method‚Äã

1 Add macro definition‚Äã

USE_IAP

2 Add the item ID‚Äã

Add the item ID to the IAPProducts.cs file.

public const string NoAds = "test.tremoveads";

public static readonly Dictionary<string, ProductType> ProductDic = new Dictionary<string, ProductType>()
{
    // ProductType.NonConsumable  only buy once. Good for one-time purchases, such as extra levels
    // ProductType.Consumable     can be purchased repeatedly. Suitable for consumable goods such as virtual currency
    // ProductType.Subscription   can be purchased and restored repeatedly. Durable goods, but for a limited period (subscription)

    { NoAds, ProductType.NonConsumable}
        
};

3 Add Purchase callback‚Äã


IAPTool.Instance.OnPurchaseDone += OnPurchaseResult;
private void OnPurchaseResult(string pID, bool pResult)
{
    if (pID.Equals(IAPProducts.NoAds) && pResult)
    {
        // After successful purchase, remove advertising
        TKGSDKManager.Instance.IsRemoveAds = true;
    }
}

IAPTool.Instance.BuyProductByID(IAPProducts.NoAds);
