账号系统

tip

在开始账号系统集成之前，请确认账号系统出现时机，并且选择所要支持的登录方式，我们提供游客登录、邮箱登录及第三方登录方式。
除了账号登录以外，我们还支持账号删除（必选）、登录方式查询（必选）、账号登出、账号绑定、强制绑定。
Touka SDK只提供登录接口，登录界面开发者可根据游戏UI风格和样式自行设计制作。

登录方式​

  1. 其他账号系统登录：游客登录、邮箱登录。
  2. 第三方账号系统登录：Google登录（仅Android）、Facebook登录、Apple登录（仅iOS）。

账号系统出现时机​

  1. 进入游戏后直接展示登录界面，玩家必须填写/勾选信息完成登录则可进入游戏。（仅首次进入展示游戏登录界面，后续使用已选方式直接登录即可。）

  2. 玩家可游客静默登录立即进入游戏体验，在合适游戏位置进行账号绑定。 （非游客账号进入游戏，不显示绑定按钮。）

说明​

  1. 单独调用游客登录、Google登录、Facebook登录、Apple登录、邮箱登录会分别返回不同的UserID。（涉及账号绑定时的情况特殊。）
  2. 注意登录是调用登录接口，登陆成功后游客账号绑定到三方账号是调用绑定账号的接口，只有调用账号绑定接口时才会把游客账号和三方账号在SDK内部做一个绑定操作；
  3. 非游客账号类型登录成功后，不提供绑定功能（即不显示绑定按钮）。 游客绑定成功后返回的登陆类型是对应绑定的三方账号类型，返回的UserID和游客登录时返回的UserID一致。

账号登录​

TKGSDKManager.Instance.Login(LoginType.LOGIN_BY_GOOGLE, (loginStatus,  loginType,  loginMsg,  userId,  token,  email,  nickName,  iconUrl) =>{

    Debug.Log("登录状态码:"+loginStatus);
    Debug.Log("登录类型:" + loginType);
    Debug.Log("登录状态信息:" + loginMsg);
    Debug.Log("用户ID:" + userId);
    Debug.Log("token:" + token);
    Debug.Log("邮箱:" + email);
    Debug.Log("显示名称:" + nickName);
    Debug.Log("头像url:" + iconUrl);
    
});

LoginType.LOGIN_BY_GUESTER (游客登录)
LoginType.LOGIN_BY_EMAIL (email登录)
LoginType.LOGIN_BY_GOOGLE (Google登录)

nickName：显示名称（返回三方账号的昵称）
iconUrl：（返回三方账号的头像url）

账号登出（可选）​

账号登出接口可在游戏切换账号时进行调用，做登出操作。SDK完成账号退出会给游戏回调。 无需求场景可不进行调用。 

TKGSDKManager.Instance.Logout();

API: 

void Logout(Action _logoutCallback = null)
NameisRequiredTypeDescription
_logoutCallbackNAction退出登录回调

登录方式查询接口（必选）​

可由SDK内部控制当前支持哪些种登录方式，返回给游戏一个登录类型List。游戏必须使用此接口，且游戏侧需根据此接口返回结果来动态调整对应方式的登录/绑定按钮的显示隐藏。

List<LoginType> list = TKGSDKManager.Instance.AvailableLoginChannelList();

API: 

List<LoginType> AvailableLoginChannelList();
TypeDescription
List <LoginType> 返回当前可用的登录类型List

账号删除（必选）​

谷歌为了增强安卓生态的透明度，扩大用户对其数据的控制权。要求支持账号登录的游戏，必须要提供入口可使用户进行账号删除操作。

  • 游戏须根据需求在游戏内添加账号删除按钮，点击账号删除按钮，会调起SDK内删除账号界面。针对此接口，游戏侧无其他必要处理。
  • 之后会托管到SDK内部处理，删除账号后给回游戏回调，然后直接退出游戏。再次打开游戏相当于一个新用户进入游戏的状态。
  • SDK删除账号界面如下图所示，长按OK按钮5s，进度条加满，弹出二级确认提示，点OK，给游戏回调，关掉游戏（SDK内处理，直接杀掉游戏进程）；点取消，关掉SDK界面回到上层/游戏界面。 
TKGSDKManager.Instance.DeleteAccount();

API: 

//DeleteStatus.DELETE_ACCOUNT_FAILED  删除失败
//DeleteStatus.DELETE_ACCOUNT_CANCEL  取消删除
//DeleteStatus.DELETE_ACCOUNT_SUCC    删除成功

void DeleteAccount(Action<DeleteStatus,string> _deleteAccountCallback = null);
NameisRequiredTypeDescription
_deleteAccountCallbackNAction <DeleteStatus,string> 删除账号回调，返回被删除的userId及状态码

账号绑定（可选）​

说明
  1. 账号绑定功能是为游戏内为游客登录时，把游客账号绑定到三方账号服务的。
  2. 只能是游客账号绑定到三方账号，且该游客账号绑定一次后不能再次绑定，如再调用绑定接口，会返回已绑定账号状态码；
  3. 游戏内可处理为绑定后不再显示绑定按钮。根据回调中的loginType为游客登录类型时才显示绑定按钮。
  4. 游客账号绑定到三方账号后返回的UserID仍为游客账号的UserID，只是在SDK内部将二者做了绑定操作。
  5. 只要游客账号没绑定过社交账号，就有绑定到社交账号的权利；如果社交账号已经被其他游客账号绑定过了，就有强制绑定的权利。这里会涉及到强制绑定的接口。
  6. 当用户使用已绑定过的社交账号进行绑定操作时，游戏可自行弹出界面列出2个账号及对应游戏进度，供玩家选择该社交账号与哪个账号进行绑定。玩家选择后，调用强制绑定接口即可。

TKGSDKManager.Instance.BindAccount(LoginType.LOGIN_BY_FACEBOOK,(bindAccountStatus,loginType,loginMsg,userId,token,email,nickName,iconUrl,userIdList)=> {

    Debug.Log("绑定状态码:" + bindAccountStatus);
    Debug.Log("登录类型:" + loginType);
    Debug.Log("登录状态信息:" + loginMsg);
    Debug.Log("用户ID:" + userId);
    Debug.Log("token:" + token);
    Debug.Log("邮箱:" + email);
    Debug.Log("显示名称:" + nickName);
    Debug.Log("头像url:" + iconUrl);
    Debug.Log("userId列表："+userIdList);

});

强制绑定（可选）​

当用户使用已绑定过的社交账号进行绑定操作时，游戏可自行弹出界面列出2个账号及对应游戏进度，供玩家选择该社交账号与哪个账号进行绑定。玩家选择后，调用强制绑定接口即可。


TKGSDKManager.Instance.ForceBindAccount("userId001",(bindAccountStatus,loginType,loginMsg,userId,token,email,nickName,iconUrl,userIdList)=> {

    Debug.Log("绑定状态码:" + bindAccountStatus);
    Debug.Log("登录类型:" + loginType);
    Debug.Log("登录状态信息:" + loginMsg);
    Debug.Log("用户ID:" + userId);
    Debug.Log("token:" + token);
    Debug.Log("邮箱:" + email);
    Debug.Log("显示名称:" + nickName);
    Debug.Log("头像url:" + iconUrl);
    Debug.Log("userId列表："+userIdList);

});

Q&A​

1、 一个Google账号可对应多个游戏账号吗？
不能。一个Google账号只能对应一个游戏账号。

2、先调用游客登录，再调用Google登录，此时游客账号和Google账号会发生绑定吗？
不会。先调用游客登录，再调用Google登录，会返回新的UserID作为新的账号进行游戏。只有调用绑定接口时，才会发生游客账号和Google账号的绑定，返回同一个UserID，即游客账号的UserID，使用同一个游戏进程。
但游客UserID和设备ID有关，此时如果设备ID变了，那前面的没进行绑定的游客账号可能就丢掉了。

3、先用游客登录，返回UserID1，有游戏进度后，选择Google账号绑定，我的UserID会变吗？
不会变。绑定后UserID仍为UserID1，只是SDK内部做了一个绑定。相当于游客账号发生绑定行为后，返回的还是原来的游客账号ID。

4、进游戏选择Google登录，有游戏进度后，重新进入游戏选游客登录，有游戏进度后，点Google账号绑定。此时会发生什么？
绑定已有进度的Google账号时，SDK会返回给游戏2个UserID，即当前游客账号UserID和已有进度的Google账号对应的UserID，游戏可自行制作界面，给玩家选择Google账号要绑定哪个游戏进度对应的UserID。然后通过强制绑定接口把玩家选定的UserID传给SDK进行和该Google账号的强制绑定。

5、对于一进游戏默认为游客登录的游戏，我先用A设备玩有进度并绑定了我仅有的Google账号，然后我又用B设备玩有进度后也要绑定我唯一的Google账号，这时候会发生什么？
这种情况同4，都是绑定已有的Google账号。

6、新生成的UserID会重复吗？
不会。新生成的UserID都是唯一的，不会重复。
