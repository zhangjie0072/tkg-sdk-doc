内购

danger
  • 需要添加 USE_IAP 宏定义
  • Unity In App Purchasing 插件版本需 4.9.3 或更高 (UnityEditor 2020.3+)

内购接入方法​

1、导入IAP插件​

导入Unity In App Purchasing插件。
Unity菜单栏 -> Window -> Package Manager -> In App Purchasing -> Install。

2、添加宏定义​

USE_IAP

3、添加内购商品​

tip

支持两种添加方式:
a. 在IAPProducts.cs中预设置商品ID
b. 通过AddProducts接口添加商品ID

注意：添加商品类型必须和后台申请类型一致，否则可能请求不到对应商品信息。
ProductType.NonConsumable 只能购买一次。适合一次性购买的商品，如额外的关卡 ProductType.Consumable 可以反复购买。适合虚拟货币等可消耗商品 ProductType.Subscription 可以反复购买和恢复。耐用品，但有效期有限（订阅）

3.1 在IAPProducts.cs中预设置商品ID。​

在 Assets/TKGSDK/Config/Scripts/IAPProducts.cs 文件中添加商品ID. 

public static class IAPProducts
{
    /// <summary>
    /// 去广告商品ID(商品ID统一小写)
    /// </summary>
    public const string NoAds = "test.tremoveads";

    public readonly static Dictionary<string, ProductType> ProductDic = new Dictionary<string, ProductType>()
    {    
         // ProductType.NonConsumable  只能购买一次。适合一次性购买的商品，如额外的关卡
         // ProductType.Consumable     可以反复购买。适合虚拟货币等可消耗商品
         // ProductType.Subscription   可以反复购买和恢复。耐用品，但有效期有限（订阅）
        { NoAds, ProductType.NonConsumable},
        //Add more product
        ...
        ...
    };
}

3.2 通过AddProducts接口添加商品ID (v3.1.2新增)​

TKGSDKManager.Instance.AddProducts(addProDic);

API:

void AddProducts(Dictionary<string, ProductType> products);
NameisRequiredTypeDescription
productsYDictionary<string, ProductType>商品ids

ex:

Dictionary<string, ProductType> ProductDic = new Dictionary<string, ProductType>()
{
    { "com.tkkk.unitysdk.demo.1",ProductType.Consumable},
    { "com.tkkk.unitysdk.demo.a1",ProductType.Consumable},
    { "com.tkkk.unitysdk.demo.a12",ProductType.Consumable}
};

TKGSDKManager.Instance.AddProducts(ProductDic);

4、监听购买回调 (v3.1.3新增)​

danger
  • 支付回调必须设置在SDK初始化之前
  • 支付回调为异步回调，回调回来时要确保游戏奖励下发正常。建议游戏侧可在支付过程中增加遮罩屏蔽用户操作 
TKGSDKManager.Instance.SetOnPurchaseDone(OnPurchaseResult);

API:

void SetOnPurchaseDone(Action<string, string, string, bool, string> onPurchaseDoneAction)
NameisRequiredTypeDescription
onPurchaseDoneActionYAction内购结果回调<苹果/谷歌三方支付订单号, 游戏侧自定义商品名称, 支付的商品ID, 是否成功支付, 游戏扩展字段>

ex:

private void OnPurchaseResult(string pOrderID, string pProductName, string pID, bool pResult, string gameExtra)
{
    if (pResult)
    {
        //add gold
    }
}
TKGSDKManager.Instance.SetOnPurchaseDone(OnPurchaseResult);

5、购买商品接口 (v3.1.2新增)​

TKGSDKManager.Instance.BuyProductByID(productID, productName, gameExtra);

API:

void BuyProductByID(string productId, string enProductName, string gameExtraParam = "")
NameisRequiredTypeDescription
productIdYstring要购买的商品ID
enProductNameYstring游戏侧自定义商品名称
gameExtraParamYstring预留游戏扩展字段 - 支付接口游戏传给SDK，SDK会在支付回调里原封返给游戏

ex:

public void BuyProduct()
{
    string productID = "com.tkkk.unitysdk.demo.a1";
    string productName = "a1";
    string gameExtra = "a1GameExtraParam";
#if USE_IAP
    TKGSDKManager.Instance.BuyProductByID(productID, productName, gameExtra);
#endif
}

6、获取本地化价格字符串接口​

TKGSDKManager.Instance.GetPriceByID(productID);

API:

string GetPriceByID(string productID)
NameisRequiredTypeDescription
productIDYstring要购买的商品ID

ex:

public void GetPriceByID()
{
    string productID = "com.tkkk.unitysdk.demo.a1";
    
    // 会返回带货币符号的价格字符串，如：'$1.99' '￥6.99'
    string price = TKGSDKManager.Instance.GetPriceByID(productID);
}

说明：遇到账号问题时，（如Googleplay无法使用中国账号）会返回空字符串，这时请使用游戏本地默认价格。

内购事件​

内购按钮展示 (v3.1.2新增)​

void LogIAPBtnShow(string productName, string productId)

NameisRequiredTypeDescription
productNameYstring游戏侧自定义商品名称
productIdYstring商品ID
GP：ProductID
iOS：SKU

调用时机： 内购按钮展示时调用。 

TKGSDKManager.Instance.LogIAPBtnShow("200golds","com.game.diamond200");

内购异常​

如遇到支付失败，请确认以下问题：

  • 支付回调必须设置在SDK初始化之前
  • 所有商品类别需和后台配置一致，消耗品、非消耗品还是订阅产品
  • 中国地区账号无法调起支付，需切换地区或使用其他地区账号
  • 网络原因，尝试切换不同vpn节点

服务端对接（可选）(v3.1.2新增)​

SDK支付成功后，游戏根据回调获取支付结果。 有服务器的游戏优先接入SDK服务器支付回调接口，支付成功后，由SDK服务器通知游戏服务器进行商品下发。单机游戏确保收到客户端支付成功回调后发放商品。

第三方游戏回调前期准备​

1、找对接方获取游戏端授权加密sercret。
2、提供游戏端回调地址给对接方进行回调地址配置。

游戏端回调地址接口说明​

1、请求方式: post
2、header授权: Authorization: secretkey
3、Body json

参数类型是否必填描述示例值
envString是环境变量（qa:测试环境; prod:线上环境）prod
platformString是平台（android/ios）android
pkgString是游戏包名com.test.demo
user_idString是用户ID0cc852594cb2dfd0381af38f687b44906
product_idString是商品IDcom.tkkk.unitysdk.demo.a1
order_idString是订单IDGPA.3340-7674-3284-90321
priceString是价格1.99
currencyString是货币单位USD
game_extra_paramString否游戏扩展字段game_extra_param_value
tsString是时间戳1692346624
signString是加密值33dasfdewffggg
body json样例（secretkey: 64f1b1fb8efadc41dcd29d5c）
{
    "env": "qa",
    "platform": "android",
    "pkg": "com.test.demo",
    "user_id": "0cc852594cb2dfd0381af38f687b44906",
    "product_id": "com.tkkk.unitysdk.demo.a1",
    "order_id": "GPA.3340-7674-3284-90321",
    "price": "1.99",
    "currency": "USD",
    "game_extra_param":"game_extra_param_value"
    "ts": "1692346624",
    "sign": "c9e91f2b7d28eec1130df9ddea5697e8"
}

4、加密方式：
sign值为：按照返回参数顺序-md5(参数对应值1+参数对应值2+参数对应值n+secretkey), +代表连接符，不参与计算

$sign = md5("qa"+"android"+"com.test.demo"+"0cc852594cb2dfd0381af38f687b44906"+"com.tkkk.unitysdk.demo.a1"+"GPA.3340-7674-3284-90321"+"1.99"+"USD"+"game_extra_param_value"+"1692346624"+$secretKey)

5、重试机制
在5个小时内最多重试10次，直到返回的http status为200且输出"success"为止。

  • 第一次 35秒
  • 第二次 40秒
  • 第三次 115秒
  • 第四次 150秒
  • 第五次 385秒
  • 第六次 755秒
  • 第七次 1500秒
  • 第八次 2610秒
  • 第九次 4230秒
  • 第十次 6850秒
