其他功能

1、游戏失去/获得焦点​

TKGSDKManager.Instance.SetGameFocusListener((_isFocus) => {
    if (_isFocus)
    {
        Debug.Log("set game focus");
    }
    else
    {
        Debug.Log("lose game focus");
    }
});

设置游戏失去/获得焦点回调，SDK每次触发到对应情境会给回来回调。
例如当广告开始播放的时候，你的游戏将会失去焦点。广告播放结束，返回游戏，你的游戏会获得焦点。

API: 

void SetGameFocusListener(Action&ltbool&gt _gameFocusAction);
NameisRequiredTypeDescription
_gameFocusActionYAction<bool>游戏焦点回调。
true:获得焦点
false:失去焦点

注：当前只对iOS生效。

TKGSDKManager.Instance.Review();

6、SDK通用回调接口​

该接口通过CommonCallbackCode来进行事件的区分。

  • 包含对firebase在线参数获取成功/失败的回调。
TKGSDKManager.Instance.SetTKGCommonCallback((code, msg) => {
    Debug.Log("common callback, code : " + code + " , msg : " + msg);
});

API: 

void SetTKGCommonCallback(Action<CommonCallbackCode, string> _commonCallback);
NameisRequiredTypeDescription
_commonCallbackYAction<CommonCallbackCode, string>CommonCallbackCode: 用来区分回调事件名称。
FirebaseOnlineParamGet_Succ: firebase在线参数获取成功。
FirebaseOnlineParamGet_Failed: firebase在线参数获取失败。

string: 消息携带参数。

7、自定义在线参数​

获取在线参数​

  • 提供远程参数获取功能，游戏中可动态获取远程参数值。
  • 在调用获取方法前，务必在以下配置文件中预先定义好参数名及默认值。

预定义参数名步骤:

  1. 添加一个枚举成员用于存储你的远程参数名:
Asset/TKGSDK/Config/Scripts/TKGParams.cs
public enum TKGParamKey
{
    RemoveAdsShowCount,
    LevelList,
    HasBlock,
    .
    .
}
  1. 将所有枚举成员及其默认值添加到OnlineParamDic:
所有自定义的在线参数的默认值都在这里定义。
Asset/TKGSDK/Config/Scripts/TKGParams.cs
  
public static class TKGParams
{
    public static readonly Dictionary<string, object> OnlineParamDic = new Dictionary<string, object>()
    {
        // key - default value
        { TKGParamKey.RemoveAdsShowCount.ToString(), 4 },    // int value    // 
        { TKGParamKey.LevelList.ToString(), "1,78,72,70,64,65,4,66" }, // string value
        { TKGParamKey.HasBlock.ToString(), 0 },     // bool value // 1:true 0:false
    };
}

  1. 参考获取远程参数的方法进行调用.
// get online param value
int onlineIntValue = TKGSDKManager.Instance.GetConfigInt(TKGParamKey.RemoveAdsShowCount);
string onlineStrValue = TKGSDKManager.Instance.GetConfigStr(TKGParamKey.LevelList);
bool onlineBoolValue = TKGSDKManager.Instance.GetConfigBool(TKGParamKey.HasBlock);

API:

NameisRequiredTypeDescription
_keyYTKGParamKey在线参数key名称
请在【产品需求文档】对应需求内获取。

8、启动屏图像设置​

注：只对iOS平台生效

SDK会根据不同渠道导出不同启动图，如有替换启动图需求，请在对应的文件夹下替换图片

CN渠道： Assets/TKGSDK/Splash/CN/


GB渠道： Assets/TKGSDK/Splash/GB/


注：启动图片名称和原启动图片名称保持一致

9、开启/关闭实名认证​

iOS平台：​

说明：​

1、CN 渠道下才会有展示实名认证，GB渠道不会展示实名认证
2、CN 渠道默认展示实名认证弹窗

修改配置：​

1、找到 CN 渠道下SDK配置文件，文件位置： Assets/Configs/CN/tkg_sdk_config.plist
2、找到文件中:
key： tkgshowauthview
value： 0 （0 关闭，1开启）

13、推送 （Android 未开放）​

1. 获取推送权限​

TKGSDKManager.Instance.RegistAPNS();

调用后弹出系统推送授权弹框

2. 注册通知​

TKGSDKManager.Instance.RegistNotification("notificationID","hello world", now.AddDays(1).ToString("yyyy-MM-dd HH:mm:ss"),1,"Title","SubTitle");

API:

void RegistNotification(string notiId, string body, string fireDate, int badge = 1, string title = "", string subTitle = "")

NameisRequiredTypeDescription
notiIdYstring注册推送的ID，具有唯一性
bodyYstring推送内容，非空，否则注册不成功
fireDateYstring推送内容触发时间，format 为 ：yyyy-MM-dd HH:mm:ss；需大于当前时间，否则注册不成功
titleNstring推送标题
subTitleNstring推送副标题

3. 移除单条推送​

TKGSDKManager.Instance.RemoveNotification("notificationID");

API:

void RemoveNotification(string notiId)<br/>
NameisRequiredTypeDescription
notiIdYstring根据推送ID移除该条推送

4. 移除全部推送​

TKGSDKManager.Instance.RemoveAllNotifications();

调用后移除全部推送

15、来源控制 （v2.2.9新增）​

说明：
1. 该接口为异步返回。List的长度决定回调方法执行的次数；如list的长度为2，则会执行两次回调方法。
2. 此方法需要在初始化SDK后调用（TKGSDKManager.Instance.InitSDK();）。

void Start() 
{
  TKGSDKManager.Instance.SetFunctionSwitchListener(new List<FunctionType>() { FunctionType.Function_wangz, FunctionType.Function_bing }, OnFunsSwitchCallback);

}

private void OnFunsSwitchCallback(FunctionType _type, bool _result)
{
  Debug.Log("[ToukaSDKDemo]：FunctionType:" + _type +"result:"+ _result);
}

API: 

void SetFunctionSwitchListener(List<FunctionType> _functionKeys, Action<FunctionType, bool> _functionSwitchCallback);
NameisRequiredTypeDescription
_functionKeysYList<FunctionType>
各功能名称集合
_functionSwitchCallbackYAction<FunctionType, bool>回调事件，返回每个功能名称及对应开关
FunctionType: 功能名称
bool: 是否开启当前功能，true - 开启 / false - 不开启
